## Quality of life

Adding `~` (for logical negation) and `else` would *really* improve quality of life, but let's do that later.

## For loops

Another thing that would really help is `for (init; cond; incr)`

However, there's a slight issue with this: while I *can* reduce it to:

```av
init;
while (cond) {
  incr;
}
```

The variable (in reality, you can have non-declarations there too, but yeah) declared by `init;` will continue to remain in scope, taking up extra memory and using up an identifier.

Although, **this** does seem to work:

```av
{
  init;
  while (cond) {
    ...
    incr;
  }
}
```

## Logical negation

Simply do the following:

```asm
cmp rax, 0
sete al
movzx eax, al
```

Note that I've made an unconventional choice and used the symbol `~` instead of `!` for logical negation.

## Else

```av
if (cond) {
  <body1>
} else {
  <body2>
}
```

Currently, plain `if` works like this:

```asm
mov rax, <cond>
cmp rax, 0
jz .after
<body1>
.after:
```

To add `else` to this, we can do this:

```asm
mov rax, <cond>
cmp rax, 0
jz .body
<body1>
jmp .after
.body:
<body2>
.after:
```

As for `else if`, let's say we had:

```av
if (cond1) {
  <body1>
} else if (cond2) {
  <body2>
} else if (cond3) {
  <body3>
} else {
  <body4>
}
```

We'd write this as:

```asm
mov rax, <cond1>
cmp rax, 0
jz .cond2
<body1>
jmp .after
.cond2:
mov rax, <cond2>
cmp rax, 0
jz .cond3
<body2>
jmp .after
.cond3:
mov rax, <cond3>
cmp rax, 0
jz .cond4
<body3>
jmp .after
.cond4:
<body4>
.after:
```

So the best way to do this would probably be to let the desugaring layer group together `if`, `else if`, ..., `else` under some node called IfElseBlock, and then just spit out the above codegen.

## Correction regarding for loops

I was very close last time, but the actual correct lowering should be:

```av
{
  init;
  while (cond) {
    {
      ...
    }
    incr;
  }
}
```

Since, otherwise, loops like this would be problematic:

```av
for (int32 i = 0; i < 5; i = i + 1) {
  int32 i = 0;
}
```

But this is an easy fix.
